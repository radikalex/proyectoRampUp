Videojuego: A Ninja Tale

Videojuego hecho en Unity que sigue la dinámica de los juego roguelike (o juegos de mazmorras) como The binding of Isaac. En este juego se generan distintas mazmorras cada vez que se juega por lo que cada partida es distinta.

El videojuego narra la historia de un ninja al cuál su maestro le otorga la misión de salvar una aldea subyugada por un samurái que está haciendo imposible la supervivencia en la aldea a través de su régimen autoritario. Nuestro protagonista deberá llegar a lo alto del castillo y derrotar al samurái, para ello deberá superar un ardúo camino derrotando a los numerosos guardias que hay.

Página Web: Héroes Cómic

Sencilla página web hecha con Angular en la que se usa JSON server para manejar los datos de los héroes y de los usuarios que tienen permitido modificarlos.

La web consta de un listado de héroes que pueden ser modificados pudiendose incluir nuevos héroes, eliminar héroes o modificar héroes ya presentes en el listado. Para añadir héroes se utilizó un formulario reactivo que tiene diversas validaciones.

Animación: Super Mario Bros

Animación hecha con Blender que trata de emular al conocido videojuego Super Mario Bros y un breve nivel de este.

Para realizar la animación se usaron distintos métodos como interpolaciones o motion capture con un fin didáctico de practicar cómo de útiles podían ser estas a la hora de crear una simulación.

La animación consta de un modelo 3D de Super Mario al que hubo que crearle un rigging, una cámara con una interpolación constante, efectos de movimiento usando el algoritmo de Catmull y la utilización de keyframes para lograr el cambio de material en la textura de Mario al coger la flor de fuego.

Simulación: Path Finding

Simulación hecha con Java en el programa 'Procesing'. En este caso de Path Finding se usó el conocido algoritmo de A*.

Este proyecto consta de un mapa de celdas blancas en las que se puede especificar una en concreto como inicio (celda verde) y otra como final (celda roja). Adicionalmente se pueden colocar celdas por las que no se puede ir clicando en ellas. Por lo que solo se puede pasar por celdas blancas para ir desde la celda verde a la roja.

La aplicación de A* en esto logra encontrar el camino más corto en un tiempo altamente eficiente. Esto se puede ver en el vídeo con las distintas pruebas que se hicieron para poner a prueba como actuaba el algoritmo.
